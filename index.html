一、Scss傳統編譯方法

1) 在終端機執行npm init -f，這步驟是為了產生一個名叫package.json的設定檔，之後需要用到。

2) 在終端機執行npm install sass來把Sass安裝到專案資料夾。安裝完後專案目錄下會出現一個node_modules資料夾來放相關檔案

3) 打開package.json，找到裡面的"test": "echo \"Error: no test specified\" && exit 1"區塊

4) 在步驟2的程式碼最後面加上一個,，接著換下一行輸入sass編譯指令

指令構造為"【指令的名稱】": "sass 【輸入的scss檔案位置+檔名】 【輸出的css檔案位置+檔名】 --watch"

例如以這次範例的檔案構造來說，指令就會是"build": "sass scss/style.scss src/css/style.css --watch"

5) 完成後，每次需編譯時只要打開終端機，執行npm run 【指令的名稱】即可

例如本次範例，就是執行npm run build

二、使用外掛程式Live Sass Compiler的編譯方法

1) 安裝外掛程式並啟用

2) 進入【檔案】->【喜好設定】->【設定】(或Ctrl+,)，叫出VS Code的設定檔

3) 打開【延伸模組】->【Live Sass Compiler Config】->【在setting.json內編輯】，叫出設定檔的本體setting.json

4) 在最後一條設定後面加上,後，換下一行，貼上以下片段並存檔：

"liveSassCompile.settings.formats":[
    {
        "format": "expanded",
        "extensionName": ".css",
        "savePath": "/src/css"
    }
]

5) 上面片段的每個設定是：

"liveSassCompile.settings.formats":[
    {
        "format": "【編譯出的css檔案的換行方式】",
        "extensionName": "【編譯出的檔案的預設附檔名】",
        "savePath": "【編譯出的檔案的存檔路徑 (路徑相對於專案的跟目錄)】"
    }
]

更多詳細設定可到Live Sass Compiler設定參考

補充) Live Sass Compiler會掃描專案底下所有的.scss檔案編譯，所以不用指定來源檔案路徑

(但編譯時，會跳過以底線_開頭命名的檔案)
